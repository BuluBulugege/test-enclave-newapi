你把 prompt（可能含敏感数据）发给 AI 网关时，正常情况下运营方能看见、能记录你发的每个字——你只能"相信"他们不这么做。这个站把"相信"换成"可验证"：用硬件 + 密码学证明服务器看不到也存不下你的数据，并让你亲眼读代码、看数据每一跳去了哪。

先认识 3 个词（用你熟悉的东西类比）

🔒
Enclave（机密飞地）
CPU 里一块硬件加密的隔离区，里面内存连服务器 root、连云厂商都看不到明文。
类比：一个连房东都打不开的保险箱，程序在里面跑。
📜
Quote（远程认证凭证）
CPU 出具、带 Intel 签名的一句话：“我这块真芯片，此刻正在跑这段确切代码”。
类比：像 HTTPS 证书，但 CPU+Intel 背书，伪造不了。
🧬
MRENCLAVE（代码指纹）
那段代码+配置的哈希，改一字节就全变。
类比：Docker digest / git commit 号 / 前端 SRI。
“它没偷看你”怎么被证明？ 一条链子，缺一环就不成立：
  1. enclave 里的代码开源、可重新编译得到同一指纹（下面"代码导读"就能逐行读）。
  2. 审计代码 ⇒ 确认它没有把正文写盘/入库/记日志的代码，且只连官方上游。
  3. Quote 由 Intel 签名，证明服务器跑的就是这份代码（指纹对得上）。
  4. ⇒ 传递性证明它做不到偷看；改代码就变指纹，下面"验证"会当场红。

②看一次请求经历了什么

下面 4 个方框是数据流经的 4 个角色。实线框=可信（enclave/官方上游/你），虚线框=不可信（new-api 主体/宿主/云厂商）。核心：prompt 永不跨过虚线框。点"发送并可视化"，逐跳看每条边界流过了什么——重点看第 2 跳。

▶
new-api 控制面不可信·只见元数据
▶
